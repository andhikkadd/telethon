# Backup & Restore Guide

This document describes how to use the backup files generated by the framework (e.g., via the `!backup` command or Web Panel UI) to restore the userbot to its original state on a new server or local development environment.

---

## 1. Understanding Backup Formats

When a backup is triggered, the system generates one of the following files in the `backups/` directory (and transmits it to your Telegram target):
- **`backup_YYYYMMDD_HHMMSS.zip.gpg`** (Recommended/Secure): A compressed ZIP archive encrypted with **AES256** using the system's `gpg` tool.
- **`backup_YYYYMMDD_HHMMSS.zip`** (Unencrypted): Generated only if GnuPG is not installed on the system and `ALLOW_UNENCRYPTED_BACKUP` is set to `true` in your `.env`.

---

## 2. Decrypting GPG Backups

If your backup file is encrypted (`.gpg`), you must decrypt it using the password set in your `.env` configuration file (**`BACKUP_PASSWORD`**) before extracting its contents.

### On Linux / macOS (Terminal)
Run the following command:
```bash
gpg --decrypt -o restored_backup.zip backup_YYYYMMDD_HHMMSS.zip.gpg
```
You will be prompted to enter the passphrase. Provide the value of `BACKUP_PASSWORD` from your original `.env` file.

### On Windows
1. Download and install **Gpg4win** from the official portal [https://gpg4win.org](https://gpg4win.org).
2. Right-click the `.gpg` file and select **Decrypt and verify** (or use the **Kleopatra** key manager interface).
3. Enter your decryption passphrase. The decrypted `.zip` file will be generated in the same directory.

---

## 3. Step-by-Step Restoration Process

Once you have obtained the decrypted `.zip` file, follow these steps to restore the application:

1. **Extract the ZIP Archive**:
   Extract `restored_backup.zip` into your new workspace. The extracted directory structure will resemble:
   ```text
   ├── data/
   │   └── bot.db              # Your SQLite database (contains groups, settings, and logs)
   ├── sessions/
   │   └── promo_userbot.session # Your MTProto Telegram session binary
   ├── docs/
   ├── main.py
   ├── requirements.txt
   └── ... (remaining source code modules)
   ```

2. **Configure Environment Variables**:
   Copy `.env.example` to `.env` in the restored directory:
   ```bash
   cp .env.example .env
   ```
   Populate the `.env` file with the exact API details (`API_ID`, `API_HASH`), admin passwords, and GPG secrets matching the old installation.

3. **Install Project Dependencies**:
   Install the required libraries:
   ```bash
   pip install -r requirements.txt
   ```

4. **Boot the Userbot**:
   Run the application entrypoint:
   ```bash
   python main.py
   ```
   Because the SQLite database (`data/bot.db`) and MTProto sessions (`sessions/`) were restored correctly, **the bot will resume operations immediately without asking for a phone number or OTP verification code**.
